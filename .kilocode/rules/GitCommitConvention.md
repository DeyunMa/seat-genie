# Git 提交信息强制规范

## 规则说明

当执行 `git commit` 命令时，必须严格遵守以下提交信息格式规范。

## 强制格式

```
<AI模型标识>：技术描述
```

## AI 模型标识对照表

- **Kimi** - 使用 Kimi K2.5 模型时
- **Claude** - 使用 Claude 系列模型时
- **MiniMax** - 使用 MiniMax 系列模型时
- **DeepSeek** - 使用 DeepSeek 系列模型时

## 格式要求

1. 必须以当前使用的AI模型标识开头
2. 后跟中文全角冒号 `：`
3. 冒号后必须有一个空格
4. 技术描述需简洁准确，概括代码变更核心内容
5. 描述类型包括：架构重构、功能新增、性能优化、Bug修复等

## 正确示例

```
Kimi：重构认证架构并迁移至JWT后端API
Claude：优化数据库查询性能减少响应时间
MiniMax：添加用户权限管理模块
DeepSeek：修复登录验证逻辑中的空指针异常
```

## 错误示例

```
更新代码                    # 缺少AI标识
Kimi: 修复bug               # 使用英文冒号
Kimi：修改文件              # 描述过于模糊
Kimi-重构架构               # 使用短横线而非冒号
```

## 执行要求

- 每次执行 `execute_command` 进行 `git commit` 时，自动检查并遵循此规范
- 根据当前模型（moonshotai/kimi-k2.5）使用 **Kimi** 作为标识
- 描述必须准确反映本次提交的核心技术变更
